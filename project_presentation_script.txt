บทสรุปโครงการ: การออกแบบสถาปัตยกรรมระบบเว็บแอปพลิเคชันแบบ 5 มิติ (5-Dimension Architecture)
========================================================================================

--- สไลด์ที่ 1: หน้าปก ---
หัวข้อ: การยกระดับความปลอดภัยและประสิทธิภาพของ Web Application ด้วยสถาปัตยกรรม 5 มิติ
คำโปรย: โซลูชันบูรณาการตั้งแต่ระดับโครงสร้างพื้นฐาน (OS), เครือข่าย, ฐานข้อมูล จนถึงระดับแอปพลิเคชัน


--- สไลด์ที่ 2: ภาพรวมและที่มาของโครงการ ---
[ หัวข้อสไลด์: ทำไมต้อง 5 มิติ? ]
• ปัญหา: ระบบแบบเดิมรันทุกอย่างรวมกันบน OS เดียวเสี่ยงต่อการโดนแฮก และกู้คืนข้อมูลได้ยาก
• ทางแก้: แยกส่วนประกอบของแอปรันบนเทคโนโลยี Container (Docker) และวางมาตรการป้องกันที่ครอบคลุม 5 จุดเสี่ยงหลัก:
  1. Access Control (การจัดการผู้ใช้งานเข้าสู่ระบบ)
  2. Infrastructure (ระบบปฏิบัติการและโครงสร้างพื้นฐาน)
  3. Web Server (เซิร์ฟเวอร์ให้บริการเว็บไซต์)
  4. Database (ความปลอดภัยของฐานข้อมูล)
  5. Security & Recovery (การสำรองข้อมูลและกู้คืน)


--- สไลด์ที่ 3: มิติที่ 1 - การจัดการผู้ใช้งานและสิทธิ์ (Access Control) ---
[ หัวข้อสไลด์: M1 - The Principle of Least Privilege ]
• เทคนิคที่ใช้: การแยกส่วน (Separation of Duties) อย่างเด็ดขาด
• รายละเอียดการทำงาน:
  1. ระดับระบบปฏิบัติการ (OS): แยกสิทธิ์บน Ubuntu Server อย่างชัดเจน
     - สร้าง User `webadmin` ให้จัดการได้แค่ไฟล์ Source Code โฟลเดอร์หน้าเว็บ
     - สร้าง User `dbadmin` ให้จัดการได้แค่โฟลเดอร์รหัสผ่านและฐานข้อมูล
     - **ผลลัพธ์**: หากโค้ดเว็บโดนเจาะ แฮกเกอร์ก็ไม่สามารถใช้สิทธิ์เอาโค้ดมารันคำสั่งพัง Database ได้
  2. ระดับแอปพลิเคชัน (App) / DB:
     - ใช้ User `webuser` แทน `root` ในการคุยกับฐานข้อมูล จำกัดสิทธิ์ป้องกันการ DROP TABLE


--- สไลด์ที่ 4: มิติที่ 2 - โครงสร้างพื้นฐาน (Infrastructure & OS) ---
[ หัวข้อสไลด์: M2 - Containerization & Isolation ]
• เทคนิคที่ใช้: Docker & Docker Compose
• รายละเอียดการทำงาน:
  - บอกลาการติดตั้งลงเซิร์ฟเวอร์โดยตรง
  - ออกแบบระบบเป็น 3 Container แยกขาดจากกัน:
    1. Web Container (PHP + Apache)
    2. Database Container (MariaDB)
    3. GUI Container (phpMyAdmin)
  - **ผลลัพธ์**: 
    - แต่ละส่วนทำงานในสภาพแวดล้อมปิด (Isolated)
    - สามารถย้ายไปรันบน Ubuntu Server หรือ Cloud เจ้าไหนก็ได้ภายในคำสั่งเดียว `docker compose up -d` 
    - ตัดขาด Network ภายนอก ฐานข้อมูลสามารถคุยกับเว็บได้แค่ในวงแลนปิดของ Docker เท่านั้น (webnet)


--- สไลด์ที่ 5: มิติที่ 3 - การจัดการ Web Server (Web Server Management) ---
[ หัวข้อสไลด์: M3 - Apache Hardening & Performance ]
• เทคนิคที่ใช้: Custom Apache Configuration overriding
• รายละเอียดการทำงาน: ปรับจูน Web Server (Apache) ให้แข็งแกร่งและทำงานเร็วขึ้น
  - **Security (vhost.conf)**: 
    - ใช้คำสั่ง `Options -Indexes` ป้องกันการเข้าดูลิสต์โฟลเดอร์ผ่าน Browser (Directory Traversal)
    - บล็อกการเข้าถึงไฟล์ซ่อน (Hidden files) เช่น `.env`, `.git`, `.htaccess` โดยตรง
  - **Performance & Masking (tuning.conf)**: 
    - ปิดการแสดงเลขเวอร์ชันของ OS และ Apache (`ServerTokens Prod`, `ServerSignature Off`) หรอกสแกนเนอร์แฮกเกอร์
    - ปรับจูน MPM Prefork เพื่อให้รองรับคนเข้าเว็บปริมาณมากได้ลื่นไหลขึ้น


--- สไลด์ที่ 6: มิติที่ 4 - การจัดการฐานข้อมูล (Database & phpMyAdmin) ---
[ หัวข้อสไลด์: M4 - Security Through Obscurity (พลางตัว) ]
• เทคนิคที่ใช้: Reverse Proxy Url Masking
• รายละเอียดการทำงาน:
  - ปกติระบบจัดการฐานข้อมูล (phpMyAdmin) มักจะใช้ Port พื้นฐาน (เช่น 8080) ทำให้โดนแฮกเกอร์ใช้เครื่องมือสุ่มเจาะหารหัสผ่านได้ง่าย (Brute Force)
  - เราทำการปิดช่องทาง Port ออกทั้งหมดทิ้ง!
  - **การแก้ปัญหา**: ส่งผ่านข้อมูลด้วยเทคนิค *Reverse Proxy* แทน
    - ซ่อนหน้าจอ phpMyAdmin ไว้ใต้เส้นทาง URL ลับ เช่น `http://your-server/db-manage-app`
    - หากแฮกเกอร์ไม่รู้พารามิเตอร์ URL ด้านหลังนี้ ก็จะไม่มีทางเข้าหน้าล็อคอิน DB ได้เลย 


--- สไลด์ที่ 7: มิติที่ 5 - การรักษาความปลอดภัย และ กู้คืนภัยพิบัติ (Security & Recovery) ---
[ หัวข้อสไลด์: M5 - Automated Cloud Disaster Recovery ]
• เทคนิคที่ใช้: Cron Job + Rclone (Google Drive Sync)
• รายละเอียดการทำงาน:
  - ข้อมูลสูญหายคือฝันร้ายที่สุดของบริษัท เราจึงทำระบบกู้คืนที่ทำงานอิสระ (Auto-Backup)
  - สร้างสคริปต์สำรองข้อมูล (mysqldump) ซึ่งจะทำงานทุกๆ ตี 2 
  - สั่งเชื่อมระบบเข้ากับ Cloud Storage (Google Drive) เพื่อดึงไฟล์ Backup ส่งออกจาก Server หลักทันที ป้องกันตึกไฟไหม้หรือเซิร์ฟเวอร์หลักพัง 
  - **ความพิเศษ**: ผู้บริหารหรือคนคุมระบบสามารถกดปุ่ม "M5: Auto-Backup" ปิด/เปิดผ่านหน้า Dashboard ของเว็บได้เองแบบ Real-Time ทำให้มีความยืดหยุ่นในการปฏิบัติงานสูง


--- สไลด์ที่ 8: บทสรุป (Conclusion) ---
[ หัวข้อสไลด์: พร้อมรับมือภัยคุกคามยุคใหม่ ]
• ระบบเว็บที่ออกแบบด้วยสถาปัตยกรรม 5 มิตินี้ เป็นมากกว่าแค่ "เว็บพิมพ์ข้อมูล" 
• มันคือแพลตฟอร์มที่ผสมผสาน Developer Operations (DevOps) และ Security เข้าด้วยกัน (DevSecOps)
• ด้วยมาตรฐานการทำงานทั้งหมดนี้ ระบบจะมีความทนทานต่อการถูกโจมตีทางไซเบอร์สูงมาก กู้คืนได้ไว ย้ายเครื่องง่าย และรองรับการขยายตัว (Scale) ในอนาคตได้อย่างเต็มประสิทธิภาพ
